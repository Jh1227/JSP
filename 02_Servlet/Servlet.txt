# 네트워크 통신
- server : 특정 서비스를 제공하는 system
  client : 서비스를 이용하는 사용자
  
  			request(요청)
  client	->		server
  			<-
  			response(응답)

# servlet
- 서버쪽에서 실행되면서 클라이언트의 요청에 따라 동적으로 서비스를 제공하는 java 클래스 입니다
- 브라우저에서 호출하면 WAS 에서 웹페이지를 동적으로 생성합니다
  > WAS 는 서버 단에서 애플리케이션이 동작 할 수 있도록 지원하는 컨테이너가 포함된 서버를 말합니다
  
# tomcat 의 servlet-api.jar 클래스 패스 설정
- 서블릿 API 는 tomcat 의 servlet-api.jar 라이브러리로 제공되므로, 클래스 패스를 설정해야 사용할 수 있습니다
- build Path -> Configure Build Path -> Libraries Tab -> Add External JARs..
  > tomcat 폴더 -> lib -> servlet-api.jar
